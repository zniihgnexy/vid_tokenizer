# Revision Checklist

Use this checklist when a draft is moving from “roughly complete” to “credible handoff”.

## Review packet expectations

A strong review packet should cover:

- overall assessment
- strongest aspects
- weakest aspects
- publication-readiness recommendation
- section-by-section issues
- concrete fixes
- open author questions

For each important issue, record:

- location
- severity: `critical`, `major`, or `minor`
- description
- suggested remediation
- whether it blocks `finalize`

## Five-pass revision default

When the deliverable is a paper bundle, use this default unless there is a good reason not to:

1. **Accuracy pass**
   - remove unsupported claims
   - repair method-description mismatch
   - verify all important numbers against source artifacts
2. **Structure pass**
   - verify file inclusion / section organization
   - ensure tables, figures, appendices, and references are reachable and named consistently
   - confirm main-text vs appendix placement still makes sense
3. **Narrative pass**
   - repair logical jumps
   - tighten transitions
   - make sure motivation, challenge, resolution, validation, and impact still match the evidence
4. **Presentation pass**
   - polish wording
   - fix citations
   - fix table and figure captions
   - fix proofing defects
5. **Final verification pass**
   - re-check the claim-evidence map
   - re-check submission checklist items
   - ensure no last-minute edits weakened accuracy

## Validation categories

Before declaring writing ready, verify at least these categories:

- content authenticity and claim accuracy
- structural completeness
- experiment/result correspondence
- citation integrity
- formatting and readability
- proofing and render quality
- packaging completeness
- handoff readiness

## Failure handling

If any `critical` issue remains:

- do not mark the bundle ready
- record the exact blocker
- either continue the revision loop or route back through `decision`
