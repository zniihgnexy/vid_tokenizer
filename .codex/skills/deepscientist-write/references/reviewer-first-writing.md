# Reviewer-First Writing

Use this reference when the draft needs to be organized around how reviewers and readers actually scan papers.

## Core assumption

Do not assume a reviewer will patiently reconstruct the paper for you.
Assume the first judgment is formed quickly from:

1. title
2. abstract
3. introduction opening
4. first decisive figure or table
5. final main-result section

## Five-minute pass

Before treating a draft as structurally sound, simulate a fast first pass and record:

- what the paper claims in one sentence
- what the main problem is
- what the current-best route fails to do
- what the method changes
- what the strongest evidence is
- whether the main claim still survives after that quick pass

Suggested durable output:

- `paper/reviewer_first_pass.md`

## Reader-first organization

Organize for reader understanding, not author chronology.

Prefer this flow:

- problem
- challenge or bottleneck in the strongest prior route
- intervention
- evidence
- boundary of the claim

Avoid writing sections that merely replay the order in which the code or experiments were built.

## First-page discipline

When the deliverable is paper-like:

- avoid wasting early page real estate on generic boilerplate
- prefer direct contributions over “This paper is organized as follows”
- if a figure or table is the fastest way to communicate the main result, place it early

## Reader questions

Every major section should answer a concrete reader question.

Examples:

- why does this problem matter now?
- why are strong existing methods still insufficient?
- what exactly changed?
- why should that change help?
- what evidence shows it helped?
- where does the claim stop?
