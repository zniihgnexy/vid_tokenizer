# Sentence-Level Proofing

Use this reference during revision and proofing, especially when the draft is understandable but still reads weakly.

## Preferred defaults

- prefer active voice when it improves clarity
- prefer strong verbs over weak verb phrases
- split overlong sentences
- remove conversational filler
- keep terminology consistent

## Common fixes

- replace vague “we want to / we try to / it can give” language with direct claims
- replace weak passive constructions when the actor matters
- shorten sentences with multiple stacked clauses
- define symbols at first use unless truly standard
- avoid symbol conflicts
- keep equation-following prose formatted consistently

## Article and phrasing checks

When unsure:

- check whether `a/an` follows pronunciation rather than spelling
- use `the` for specific referents, not by default
- verify unstable phrase choices with exact-phrase search or comparable corpora

## Citation prose

Keep citation style consistent:

- author-as-subject style when the sentence is about the authors’ argument
- parenthetical style when the citation is support, not the grammatical subject

## Proofing output

Suggested durable output:

- `paper/proofing/language_issues.md`

Record:

- location
- issue type
- old wording
- revised wording
- whether the fix is still pending
